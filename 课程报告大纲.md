## 2. 项目背景与目标

本项目开发了一款基于 Qt 框架的 FPGA 交互软件，用于在开发过程中调试和测试 FPGA 设计。软件通过图形化界面与 FPGA 进行数据交互，本次工作新增了数据捕获和数值输入两类组件，丰富了交互手段。

**开发环境：**
- 框架：Qt5/Qt6
- 语言：C++
- 目标平台：FPGA 开发板

## 3. 系统设计

### 3.1 整体架构

软件采用组件化设计思想，每个功能单元封装为独立组件。主要组成部分：

```
┌─────────────────────────────────────────┐
│            MainWindow                    │
│  ┌─────────────────────────────────┐     │
│  │      Component Palette          │     │
│  │  (输入/输出组件选择)            │     │
│  └─────────────────────────────────┘     │
│  ┌─────────────────────────────────┐     │
│  │      Canvas                     │     │
│  │  (组件实例化与连接)             │     │
│  └─────────────────────────────────┘     │
│  ┌─────────────────────────────────┐     │
│  │      FPGA Communication         │     │
│  │  (数据读写)                     │     │
│  └─────────────────────────────────┘     │
└─────────────────────────────────────────┘
```

### 3.2 组件分类

| 类型 | 说明 | 示例 |
|------|------|------|
| 输入组件 | 产生数据发送给 FPGA | Switch, Button, DecimalInput |
| 输出组件 | 接收并显示来自 FPGA 的数据 | LED, LCD, DataCapture |

### 3.3 数据流

```
输入组件 → getWriteData() → FPGA
FPGA → processReadData() → 输出组件
```

## 4. 组件设计与实现

### 4.1 DataCapture 数据捕获组件

#### 4.1.1 功能概述

DataCapture 组件用于从 FPGA 接收数据。当 FPGA 输出的 CAPTURE 信号为高时，组件捕获数据端口的值，并以多种进制格式显示。

#### 4.1.2 支持的数据类型

| 组件名称 | 数据宽度 | 说明 |
|----------|----------|------|
| DataCapture8 | 8 bit | 捕获 8 位整数 |
| DataCapture16 | 16 bit | 捕获 16 位整数 |
| DataCapture32 | 32 bit | 捕获 32 位整数 |
| DataCaptureFloat | 32 bit | 捕获 32 位浮点数 |

#### 4.1.3 端口定义

```
┌─────────────────────────────┐
│  DataCapture8               │
├─────────────────────────────┤
│  Input:                     │
│    - CAPTURE (1 bit)        │  ← 捕获触发信号
│    - DOUT0 ~ DOUT7 (8 bit)  │  ← 数据输出端口
└─────────────────────────────┘
```

**端口说明：**
- `CAPTURE`：触发信号，为高时执行数据捕获
- `DOUT0`~`DOUT7`：数据端口，DOUT0 为 LSB，DOUT7 为 MSB

#### 4.1.4 核心实现

**数据捕获逻辑（DataCapture8Component.cpp）：**

```cpp
void DataCapture8RawComponent::processReadData(QQueue<uint64_t> &read_queue) {
  if (read_queue.isEmpty()) {
    return;
  }

  // 获取 CAPTURE 信号端口索引
  auto capture_port_index = output_ports_[0].pin_index - 1;

  // 检查最新的采样值
  auto latest_value = read_queue.back();

  // 判断 CAPTURE 信号是否为高
  bool capture_high = (latest_value >> capture_port_index) & 0x1;

  if (capture_high) {
    // 捕获 8 位数据
    int data = 0;
    for (auto i = 0; i < 8; ++i) {
      auto port_index = output_ports_[i + 1].pin_index - 1;
      data |= ((latest_value >> port_index) & 0x1) << i;
    }
    captured_value_ = data;
    has_captured_ = true;
    updateDisplay();
  }
}
```

**浮点数捕获（DataCaptureFloatComponent.cpp）：**

```cpp
void DataCaptureFloatRawComponent::processReadData(QQueue<uint64_t> &read_queue) {
  // ...
  if (capture_high) {
    // 捕获 32 位浮点数据
    quint32 data = 0;
    for (auto i = 0; i < 32; ++i) {
      auto port_index = output_ports_[i + 1].pin_index - 1;
      data |= (((quint64)latest_value >> port_index) & 0x1) << i;
    }
    // 将比特模式转换为浮点数
    memcpy(&captured_value_, &data, sizeof(captured_value_));
    has_captured_ = true;
    updateDisplay();
  }
}
```

#### 4.1.5 GUI 显示

组件界面包含以下显示元素：
- **状态标签**：显示 "Waiting..." 或 "Captured"
- **十进制显示**：如 `Dec: 255`
- **十六进制显示**：如 `Hex: 0xFF`
- **二进制显示**：如 `Bin: 11111111`

### 4.2 DecimalInput 十进制输入组件

#### 4.2.1 功能概述

DecimalInput 组件提供图形化界面，允许用户输入十进制数值并发送到 FPGA。适用于需要手动设置测试数据的场景。

#### 4.2.2 支持的数据类型

| 组件名称 | 数据宽度 | 输入范围 |
|----------|----------|----------|
| DecimalInput8 | 8 bit | 0 ~ 255 |
| DecimalInput16 | 16 bit | 0 ~ 65535 |
| DecimalInput32 | 32 bit | 0 ~ 4294967295 |
| DecimalInputFloat | 32 bit | 浮点数 |

#### 4.2.3 端口定义

```
┌─────────────────────────────┐
│  DecimalInput8             │
├─────────────────────────────┤
│  Output:                   │
│    - DIN0 ~ DIN7 (8 bit)   │  ← 数据输入端口
└─────────────────────────────┘
```

**端口说明：**
- `DIN0`~`DIN7`：数据端口，DIN0 为 LSB，DIN7 为 MSB

#### 4.2.4 核心实现

**数据发送逻辑（DecimalInput8Component.cpp）：**

```cpp
uint64_t DecimalInput8RawComponent::getWriteData() const {
  uint64_t data = 0;
  for (auto i = 0; i < 8; ++i) {
    // DIN0 是 LSB (pin_index 0), DIN7 是 MSB (pin_index 7)
    // current_value_ 的第 i 位发送到第 i 个输入端口
    data |= ((current_value_ >> i) & 0x1) << input_ports_[i].pin_index;
  }
  return data;
}
```

**浮点数发送（DecimalInputFloatComponent.cpp）：**

```cpp
uint64_t DecimalInputFloatRawComponent::getWriteData() const {
  // 将浮点数转换为 32 位比特模式
  quint32 float_bits;
  memcpy(&float_bits, &current_value_, sizeof(float_bits));

  quint64 data = 0;
  for (auto i = 0; i < 32; ++i) {
    data |= ((float_bits >> i) & 0x1) << input_ports_[i].pin_index;
  }
  return data;
}
```

#### 4.2.5 GUI 显示与交互

组件界面包含：
- **数值显示**：十进制、十六进制、二进制三种格式
- **输入框**：直接编辑数值（带范围校验）
- **递增/递减按钮**：+1/-1 调整数值（8/16/32 位版本）

### 4.3 ArrayPortMappingSettingsDialog 端口映射设置

#### 4.3.1 功能概述

当 FPGA 设计中使用数组形式的端口（如 `data[0]`, `data[1]`, ...）时，手动逐一映射非常繁琐。ArrayPortMappingSettingsDialog 提供批量自动映射功能。

#### 4.3.2 使用场景

假设 FPGA 约束文件定义如下端口：
```
Input:  data[0], data[1], data[2], ..., data[31]
Output: result[0], result[1], ..., result[7]
```

组件内部端口为：
```
Input:  DIN0, DIN1, DIN2, ..., DIN31
Output: DOUT0, DOUT1, ..., DOUT7
```

通过指定数组前缀 `data` 和 `result`，可一键完成所有端口的映射。

#### 4.3.3 核心实现

**自动映射逻辑：**

```cpp
void ArrayPortMappingSettingsFeatureWidget::onAutoMapClicked() {
  // 从约束文件读取可用端口
  rabbit_App::ports::PortsFileReader reader;
  reader.readFromFile(constraint_path);

  QString input_prefix = input_array_combobox_->currentText();
  QString output_prefix = output_array_combobox_->currentText();

  auto raw_component = component_->rawComponent();
  auto &inputs_vec = raw_component->inputPorts();
  auto &outputs_vec = raw_component->outputPorts();

  // 映射输入端口
  for (auto &port : inputs_vec) {
    // 从端口名提取索引，如 "DIN5" -> 5
    QRegularExpression regex(R"(DIN(\d+))");
    QRegularExpressionMatch match = regex.match(port.name);
    if (match.hasMatch()) {
      int index = match.captured(1).toInt();
      QString array_port_name = QString("%1[%2]").arg(input_prefix).arg(index);

      // 查找并映射
      for (auto &hdl_port : reader.inputs()) {
        if (hdl_port.name == array_port_name) {
          parent_dialog_->setHdlPortName(port.name, hdl_port.name);
          break;
        }
      }
    }
  }
  // 类似处理输出端口...
}
```

## 7. 总结与展望

### 7.1 工作总结

本次工作新增了 8 个数据交互组件：

- **DataCapture 系列**：8/16/32 位整数和浮点数据捕获
- **DecimalInput 系列**：8/16/32 位整数和浮点数据输入
- **ArrayPortMappingSettingsDialog**：批量端口映射工具

这些组件丰富了 FPGA 交互软件的数据交互能力，使得调试过程更加便捷。

### 7.2 可改进方向

1. **添加更多数据类型支持**：如带符号整数、定点数
2. **波形显示**：为 DataCapture 组件添加时序波形显示功能
3. **批量数据记录**：支持将捕获的数据导出为 CSV 格式
4. **组件配置保存**：将组件设置持久化到项目文件
